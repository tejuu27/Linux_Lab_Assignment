IO INVESTIGATION REPORT

1. File Creation and Redirection
Commands:
echo "System log entry 1" > log.txt
echo "System log entry 2" >> log.txt

Observations:
The ">" operator creates or overwrites a file, while ">>" appends data to an existing file without removing previous content. This demonstrates basic output redirection in Linux.

2. File Content Verification
Command:
cat log.txt

Observation:
The file contains both log entries, confirming successful write and append operations.

3. File Descriptor Concept
Command:
ls -l /dev/fd

Observation:
File descriptors 0, 1, and 2 represent standard input, output, and error streams respectively. Linux treats these streams as files.

Note:
On macOS, /proc filesystem is not available. File descriptors are demonstrated using /dev/fd instead.

4. Error Redirection
Command:
ls nonexistent_file 2> error.txt

Observation:
The error message generated by the command was redirected into error.txt using stderr redirection (2>).

5. Error File Content
Command:
cat error.txt

Observation:
The file contains the error message indicating that the file does not exist.

6. Resource Limits
Command:
ulimit -a

Observation:
Displays system-imposed limits on resources such as file size, processes, and memory usage. These limits help ensure system stability.

7. Conclusion
Linux I/O system uses file descriptors to manage input, output, and error streams. Redirection operators allow control over where output is sent, and system resource limits ensure efficient resource usage.
